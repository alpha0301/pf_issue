# DB 서버 응답시간 분석

## 개요
- 특정 SQL의 느린 성능 확인
- 처리시간이 짧은 SQL의 발생 빈도 확인  
 -> 어플리케이션 로직 분석도 같이 확인 필요

## 도구
- DB: db 클라이언트
- 스택: pstack

## 특정 SQL 분석
- 실행계획을 통해 원하는 구조대로 동작하는지 확인(index)
- 락 대기 유무 확인
  - 수행중인 SQL간의 락 트리 확인

## 짧은 SQL의 발생 빈도 분석
- 수행 통계 확인
- 동작 중인 세션 이상 확인(대기 이벤트)
- 시스템 전반에 대한 운영 통계 확인(오라클 AWR)

## 어플리케이션 로직 분석
- 집합 처리
- 캐시
- SQL 통합
- 불필요한 쿼리 제거
